Client Feedback


Our clients for Anima v3 are UCL, GOSH/ROCHE, and the International Alliance of ALS/MND Associations. Our main point of contact was Professor Dean Mohammedally, who facilitated the communication and gathered requirements from these stakeholders. Throughout the project, the professor refined our project’s objectives and he has provided us with essential resources including contact with prominent researchers from Intel and UCL. The professor particularly commended our achievement in compiling the backend using PyOxidizer, allowing Anima to function as a standalone application. We are equally proud of Anima v3’s significant improvement from its predecessor as it now has a wealth of additional functionalities, a modern UI and the ability to run as a standalone application.

Critical Evaluation


User Experience

In this project, enhancing the user experience for offline voice banking was paramount. Our design process was thorough, involving multiple stages of development and user feedback to refine the user interface.

Initially, in the Human-Computer Interaction (HCI) phase, we drew two sketches and developed an initial prototype, subsequently iterating again to create a final prototype design in Figma. This iterative design process, enriched by continuous user feedback, allowed us to make progressive enhancements to the user experience.

The transition from the Microsoft Foundation Class (MFC) to .NET for the front-end was a strategic decision. MFC, while robust, often results in outdated and complex interfaces. By adopting .NET, we capitalised on its modern framework capabilities to craft a more contemporary and intuitive user interface. This move to .NET not only facilitated the development of an aesthetically pleasing design but also streamlined the development process, making it more efficient due to .NET’s IDE and comprehensive libraries which simplified coding and testing.

After implementing the .NET design, feedback was solicited from Teaching Assistants (TAs) and Dr. Yun Fu. The initial colour palette, noted for its brightness, was critiqued for lacking professionalism by the TA and pseudo-users. Dr. Yun Fu, as a first-time user, highlighted navigational difficulties across the three main windows. Responding to these insights, we conducted additional design iterations, selecting more subdued colours and integrating navigational buttons to quickly transition between the main windows.

Our design process for the user interface (UI) was comprehensive at each stage, yet there remains room for improvement, as no design is flawless. For instance, replacing the buttons with a closable side menu could streamline navigation to the main windows and reduce clutter to enhance the overall aesthetic and functionality of the interface.


Functionality

Anima V3 is a voicebanking application at it's core therefore the most important functionality of the application is a seamless way of users to bank their voice as well as to speaked with that banked voice. We have conducted rigorous unit and integration testing for these funtionalities, and ensured that they are working as intended. We have also provided additional functionalities such as the ability to read jpeg, png and pdf files (optical character recognition - OCR) and speak the text in them, as well as the ability import .wav files and register an digital voice file (.animaprofile). OCR technology was already included in AnimaV2 however it was inaccurate and unreliable. We have switched the OCR library to easyocr in Anima V3 and made it more accurate and reliable.

Moreover, Anima V3 many failsafe technologies to ensure that the application is robust and reliable. For example, the user always has the option to opt of out the text-to-speech process. This is very crucial as the user might accidentally select the wrong file when speaking from a file or enter a wrong text when speaking from text. We have also added a management system for the user's voicebank, giving the user control and customizability over their voicebank. Overall, we believe that Anima V3 provides it's users with a robust and reliable set of functionalities that are easy to use and understand. It's very easy to navigate and the functionalities are clear and easy to use. Hence, we think that Anima V3 is a huge success in terms of functionalitiy.


Accessibility

AnimaV3's primary goal is to enable individuals with motor neuron diseases (MND) to communicate using their own voice, making accessibility a central aspect of the application's design. To enhance accessibility, we have undertaken significant steps to simplify the deployment process and broaden language support, while also improving the accuracy of dialect representation.

Initially, the use of Docker in AnimaV2 presented a substantial obstacle for users not accustomed to technological complexities. Recognising this, we removed Docker in AnimaV3, transforming it into a standalone application compatible with any Windows 8+ device. This change significantly eases the installation process, making the application as user-friendly as downloading and using an app from the app store.

In addition to technical improvements, AnimaV3 now supports multiple languages, specifically Portuguese and French alongside English. This expansion is complemented by specialised texts for voice capture, tailored to each language, facilitating a more authentic and inclusive voice banking experience.

The introduction of EasyOCR technology in AnimaV3 has been a game-changer, allowing for seamless conversion of printed text to speech and eliminating the need for manual text entry. For example, if the user would like to read a PDF book, all they have to do is upload the file rather than copy and paste the entire text.

Our acknowledgement of accent bias in the previous versions led to a thoughtful approach in AnimaV3. Although replacing the Coqui TTS model was beyond our project's timeframe, we opted for a solution that involved using longer and more diverse texts to capture the voice profile of the user. This method aims to better capture the unique accents and dialects of users, despite being a temporary measure that slightly extends the time it takes to set up your voice.

In summary, AnimaV3 represents a major stride towards creating a more accessible voice banking application. It stands out for being free, easy to set up, supporting multiple languages, and accommodating various accents. Although there is always room for enhancement, such as extending compatibility to more operating systems, adding additional languages, and refining the TTS model, AnimaV3 has undoubtedly set a new standard in the field of accessible offline voice-banking applications.


Stability

Anima V3 is a standalone application that can be installed and run on any Windows operating system after Windows 8. We have conducted rigorous compatability testing to ensure that Anima V3 is compatible with all Windows operating systems after Windows 8. The details of the compatability testing counducted can be found under the Testing page. We have also conducted performance/stress testing to ensure that Anima V3 is stable under different loads. We have tried using the application with different lengths of text and different types of text and have found that Anima V3 is stable under different loads.

We have also tested the edge cases of Anima V3 to make the application more stable. We have tried to bank a voice with a username that is already taken, speak from a file that is not supported, saying the same text over and over again, reading the same file and many more. Some of the edge cases were not considered during initial development and they were promptly fixed as they were discovered. We have then ensured the stability of our application bu conducting user testing and let the pseudo-users use the application as they please to see if there were any other bugs or edge cases that we have missed. We are confident Anima V3 is a hard-to-break and very stable application.


Efficency

With Anima V3 efficency of our application has been greatly improved. By removing Docker from Anima V3, we have made the application much more efficent as Docker is a resource-heavy application that always had to be running in the background for AnimaV2. We have also made the application more efficient by adding a management system for the user's voicebank, making the voice-banking experience much easier and efficient. Lastly, we optimized the codebase by removing unnecessary checks for language and users in the Anima API.


Compatability

Anima V3 is a standalone application with a .NET frontend and Python backend. It uses Python 3.9.11 and .NET 4.7.2. Since Anima V3 is using .NET, it is only compatible with Windows OS, meaning that users with other operating systems such as MacOS or Linux cannot use Anima V3, while Anima CLI (without GUI) can still be used. It also means that Anima V3 is not compatible with any Windows operating system before Windows 8, as Python 3.9.11 is not supported on those operating systems. We have still conducted compatability testing to ensure that Anima V3 does not work with Windows XP and Windows 7 and works with Windows 8.1, Windows 10 and Windows 11. The details of the compatability testing counducted can be found under the Testing page.


Maintainability

Anima V3 is designed with maintainability in mind adhering to the modular design and code structure principle. It has a clear separation of concerns between the frontend and backend. The code documentation is clear and concise, and the code is well structured. The documentation and the code structure can be improved upon to make it even easier for future developers to understand and maintain the code.


Project Management

The project has been managed relatively well, with each team member taking responsibility for different aspects of the project. We had regular team meetings, closely followed our Gantt chart, as well as weekly client meetings on Tuesdays to ensure we were on track to finish what we needed. The team has been good at communication and collaboration, keeping each other on track and helping each other out when needed. We have also been good at keeping our client informed of our progress and any issues we have encountered.

Future Work


Anima V3 has been a huge step forward. We believe the work done with Anima V3 can change the lives of many people for the better. We are also aware that voice-banking is a very serious issue and there is still improvements to be amde. Here are some of the future work that can be done to improve Anima V3:

  1. We are currently using a multi-language model from CoquiTTS engine. For a better sound quality, a different model for each language can be used/trained. Or the text-to-speech engine can be changed to another open-source engine with a better sound quality.

  2. Additional dialect and language support can be added. Currently this is problem is being tackled through diverse voice-banking texts curated for users however the banked voices are still biased towards the American accent and need a better solution.

  3. When banking a voice, highlight the text the user is reading to enhance the voice-banking experience

  4. Record each page separately and create the animaprofile from the recordings from all pages. Also let the user know when the recording starts and stops clearly.

  5. Make the sepeartion between windows more clear. Highlight the window the user is currently on and grey out the other two to enhance user-friendliness.
