Critical Evaluation


User Experience

One of our main objectives for this project was to enhance the user experience for voicebanking therefore user experience was a main priority for us. By switching to .NET from MFC, we have managed to create a more modern and user-friendly interface. We have also made sure that the application is easy to navigate and the functionalities are clear and easy to use. We conducted user testing with pseudo-users, friends and TA's and have gathered feedback on the design and functionality of the application. We have changed our design before, during and after development to ensure that we accomodate to the user's needs as much as possible. We believe that our efforts in this area has been a real success, and the feedback we have received from Dr. Yun Fu and our clients supports our conclusion.


Functionality

AnimaV3 is a voicebanking application at it's core therefore the most important functionality of the application is a seamless way of users to bank their voice as well as to speaked with that banked voice. We have conducted rigorous unit and integration testing for these funtionalities, and ensured that they are working as intended. We have also provided additional functionalities such as the ability to read jpeg, png and pdf files (optical character recognition - OCR) and speak the text in them, as well as the ability import .wav files and register an digital voice file (.animaprofile). OCR technology was already included in AnimaV2 however it was inaccurate and unreliable. We have switched the OCR library to easyocr in AnimaV3 and made it more accurate and reliable.

Moreover, AnimaV3 many failsafe technologies to ensure that the application is robust and reliable. For example, the user always has the option to opt of out the text-to-speech process. This is very crucial as the user might accidentally select the wrong file when speaking from a file or enter a wrong text when speaking from text. We have also added a management system for the user's voicebank, giving the user control and customizability over their voicebank. Overall, we believe that AnimaV3 provides it's users with a robust and reliable set of functionalities that are easy to use and understand. It's very easy to navigate and the functionalities are clear and easy to use. Hence, we think that AnimaV3 is a huge success in terms of functionalitiy.


Accessibility

Our application is giving the ability to speak with their unique voice to people who can no longer do so on their own. It is an accessibility app that makes speech accessible to people with motor neuron diseases. Therefore, it is crucial that the application is accessible to as many people as possible. Hence the following are the steps we took to ensure that AnimaV3 reaches as many people as it possibly can. Firstly, the predecessor of AnimaV3 used a Docker container to run the application. This was a huge barrier for many users as Docker is not easy to use for people who are not familiar with it or computers. We have removed Docker from AnimaV3 and made it a standalone application that can be installed and run on any Windows operating system after Windows 8. This makes AnimaV3 much more accessible to people who are not coming from technology backgrounds.

Moreover, we added additional language support. The multi-language model we are currently using supports English, French and Portugese languages and we have included all these languages in AnimaV3. We have also insured that files in these languages can also be read with OCR and spoken with text-to-speech. This means AnimaV3 is much more accessible than it's competitors who are only tailoring to the English speaking audience as that is the biggest audience for voice-banking. AnimaV3 is completely free doesn't need any financial incentive make it accessible to non-English speakers. With the upcoming Anima versions, we believe that many more languages can be added and Anima can be accessible to a much wider audience.

Furthermore, from our predecessors, we know that AnimaV3 has a bias towards American accents for English. We have also noticed that for French the bias is for the Parisian accent and for Portugese the bias is for the Brazilian accent. We want all accents to be represented by AnimaV3 to preserve our user's unique voice and identity. As we didn't have the time to implement a new model, we have done the next best thing. We have picked texts that are long and diverse enough third-party the model can represent the user's accent well enough. Although this is not permanent solution and slightly increases the time for banking a voice, we believe that it is a good solution for representing one's accent.

Overall, we believe that AnimaV3 is a huge success in terms of accessibility. It is much more accessible than it's predecessor and it's paid competitors. It is free, easy to set-up and use, supports multiple languages and accents. AnimaV3 is one of the most accessible voice-banking applications out there and we are very proud of this achievement.


Stability

AnimaV3 is a standalone application that can be installed and run on any Windows operating system after Windows 8. We have conducted rigorous compatability testing to ensure that AnimaV3 is compatible with all Windows operating systems after Windows 8. The details of the compatability testing counducted can be found under the Testing page. We have also conducted performance/stress testing to ensure that AnimaV3 is stable under different loads. We have tried using the application with different lengths of text and different types of text and have found that AnimaV3 is stable under different loads.

We have also tested the edge cases of AnimaV3 to make the application more stable. We have tried to bank a voice with a username that is already taken, speak from a file that is not supported, saying the same text over and over again, reading the same file and many more. Some of the edge cases were not considered during initial development and they were promptly fixed as they were discovered. We have then ensured the stability of our application bu conducting user testing and let the pseudo-users use the application as they please to see if there were any other bugs or edge cases that we have missed. We are confident AnimaV3 is a hard-to-break and very stable application.


Efficency

With AnimaV3 efficency of our application has been greatly improved. By removing Docker from AnimaV3, we have made the application much more efficent as Docker is a resource-heavy application that always had to be running in the background for AnimaV2. We have also made the application more efficient by adding a management system for the user's voicebank, making the voice-banking experience much easier and efficient. Lastly, we optimized the codebase by removing unnecessary checks for language and users in the Anima API.


Compatability

AnimaV3 is a standalone application with a .NET frontend and Python backend. It uses Python 3.9.11 and .NET 4.7.2. Since AnimaV3 is using .NET, it is only compatible with Windows OS, meaning that users with other operating systems such as MacOS or Linux cannot use AnimaV3. It also means that AnimaV3 is not compatible with any Windows operating system before Windows 7 as Python 3.9.11 is not supported on these operating systems. We have still conducted compatability testing to ensure that AnimaV3 does not work with Windows XP and Windows 7 and works with Windows 8.1, Windows 10 and Windows 11. The details of the compatability testing counducted can be found under the Testing page.


Maintainability

AnimaV3 is designed with maintainability in mind adhering to the modular design and code structure principle. It has a clear separation of concerns between the frontend and backend. The code documentation is clear and concise, and the code is well structured. The documentation and the code structure can be improved upon to make it even easier for future developers to understand and maintain the code.


Project Management

The project has been managed relatively well, with each team member taking responsibility for different aspects of the project. We had regular team meetings, closely followed our Gantt chart, as well as weekly client meetings on Tuesdays to ensure we were on track to finish what we needed. The team has been good at communication and collaboration, keeping each other on track and helping each other out when needed. We have also been good at keeping our client informed of our progress and any issues we have encountered.
