UI Design Principles

Anima V3 is designed with simplicity and user-friendliness in mind. Here are some user interface design principles we had in mind when designing Anima V3.

Consistency and Visibility

The colours and button shapes, as well as fonts, are consistent in Anima V3.

We chose the following colours:
#214edf (Blue) #10c605 (Green) #f40205 (Red) #fcf7e4 (Beige)
This provides a comfortable colour pallette for the user.

We used the Microsoft Segoe UI font family.

Simplicity

Anima V3 should be simple and easy to navigate. There are only three pages the user mainly interacts with. We made this consideration for our target users (patients suffering from motor neuron diseases), this minimises the effort needed to learn how to use Anima V3. This should be the case in general no matter the target user anyway.

Accessibility and Visibility

Anima V3 is designed with bright colours with high contrast to the background, especially suiting the needs of our target user group mentioned above. It is also generally good practice to consider colour-blind users.

Error Handling and Prevention

Anima V3 provides informative error pop-up windows. This informs the user of what user errors are encountered.

Design Documentation

Sketch 1:

This is based off Anima V2's old design. It features rectanglular blocks for buttons and other UI elements. It has the basic pages for reading text to bank your voice, and typing text to speak with your voice. There is also a read from pre-loaded books page which was later scrapped and replaced by the OCR funcitonality.
This design was not adopted since we are trying to innovate the UI for Anima V3, so we came up with Sketch 2.

Sketch 2:

This is based off the UI of I Will Always Be Me, an online book for voicebanking developed by Dell and Intel. We conducted research on their UI design and website design, and came to the conclusion that what made their designs work was curves and a minimalistic look. Read more about our research on this online book here.
The new sketch features UI elements separated by lines for the miniamlistic look. There are rounded buttons compared to the rectangular ones in Sketch 1. The page counter side bar design is also first seen here. The voice bank manager page is in this sketch which would be adopted.
This design was adopted as we like how it gives Anima V3 a new look. It was also supported by our client.

First Prototype:

This is based off Sketch 2. The colours used here are loosely based off the aforementioned I Will Always Be Me project. There are only 3 main pages which will later become the main pages of the final product.
There is not much to be said about the first prototype, it is a rough UI design that was not hand drawn and it served as a simulation to what Anima V3 will actually look like on a Windows machine.

Improved Prototype:

This is where it starts to resemble our final product. We conducted more user research to gather their opinions on how to improve and finalize our design.
The most noticeable change is we finalized the font to be the Microsoft Segoe UI font family, with white text on coloured buttons. The text on buttons are also more informative and specific, better telling the user what they are for. A even more minimalistic design is used with the images used for decoration.

Side note:
Notice that at this stage there were talks about integrating Anima V3 with the rest of MotionInput, a touchless computing project led by UCL, hence the design with the MotionInput keyboards. This didn't carry over to the final product as our client no longer required us to integrate with MotionInput.

Final UI Design:

There are not many changes with this from the improved prototype.
There is now a language dropdown menu since we added multi language support. We also gathered feedbck from first time users that it is better to add navigation buttons between the three main pages at the top, so we added them. The MotionInput keyboard functionality was also removed as it was no longer needed.
